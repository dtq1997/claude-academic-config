# 多智能体协同规则 - Archon 系统

## 智能体角色分工

### Claude Opus 4.6
**角色 1: 规划智能体 (Plan Agent)**
- 全局视角任务分解
- 瓶颈识别和干预策略选择
- 在干净上下文中运行

**角色 2: Lean 智能体**
- 执行形式化代码生成
- 处理编译错误和技术细节
- 与 Mathlib 深度交互

### Gemini 3.1 Pro Preview
**角色: 非正式智能体 (Informal Agent)**
- 数学推理和证明生成
- 填补证明 gap
- 提出替代证明策略

### GPT-5.4 (可选)
**角色: 备用非正式智能体**
- 当 Gemini 失败时的后备方案
- 交叉验证关键数学判断

## 协同工作流

### 标准循环

```
1. [规划智能体] 分析整体任务，生成工作计划
   ↓
2. [Lean 智能体] 执行具体形式化任务
   ↓
3a. 成功 → 继续下一个任务
3b. 失败 → 返回诊断信息给规划智能体
   ↓
4. [规划智能体] 分析失败原因，选择干预策略：
   - 策略 A: 调用 [非正式智能体] 生成详细证明
   - 策略 B: 指示 [Lean 智能体] 分解任务
   - 策略 C: 调用 [非正式智能体] 提出替代路线
   ↓
5. 返回步骤 2，带上新的指导
```

### 并行模式

当证明义务彼此独立时：

```
[规划智能体] 识别独立任务 T1, T2, T3
   ↓
[Lean 智能体 #1] ← T1
[Lean 智能体 #2] ← T2  (并行执行)
[Lean 智能体 #3] ← T3
   ↓
[规划智能体] 汇总结果，处理依赖关系
```

## 上下文管理

### 规划智能体上下文
**包含：**
- 整体项目结构
- 所有模块的状态摘要
- 失败历史和已尝试策略
- 非正式证明全文

**不包含：**
- 详细的 Lean 代码
- 编译错误细节
- 库 API 文档

### Lean 智能体上下文
**包含：**
- 当前任务的聚焦指令
- 相关的 Lean 文件
- 编译诊断信息
- LeanSearch 检索结果

**不包含：**
- 其他模块的代码
- 全局策略讨论
- 失败尝试的完整历史

### 非正式智能体上下文
**包含：**
- 具体的数学问题陈述
- 已有的非正式证明片段
- 形式化障碍描述

**不包含：**
- Lean 代码
- 编译错误
- 库 API 细节

## 通信协议

### 规划智能体 → Lean 智能体

```markdown
## 任务
证明引理 `foo_bar_lemma`

## 位置
文件: `src/main.lean`, 行 42-50

## 策略
使用归纳法，先证明 base case，再证明 inductive step

## 支持材料
[非正式智能体生成的详细证明]

## 约束
- 不要使用 `sorry`
- 优先使用 Mathlib 现有引理
- 如果卡住超过 30 分钟，报告失败
```

### Lean 智能体 → 规划智能体

```markdown
## 状态
失败

## 问题
无法证明 `inductive step`，缺少关于 `X` 的引理

## 已尝试
- 搜索 Mathlib: 未找到相关结果
- 尝试直接证明: 证明项过于复杂（>500 行）
- 尝试 `omega` 策略: 超时

## 建议
需要将 inductive step 分解为 2-3 个子引理
```

### 规划智能体 → 非正式智能体

```markdown
## 请求类型
详细证明生成

## 定理陈述
对于所有正整数 n，如果 P(n) 成立，则 P(n+1) 成立

## 上下文
- P(n) 的定义: [...]
- 已知引理: [...]

## 要求
- 逐步推导，不跳过任何步骤
- 明确指出使用的每个引理
- 避免使用 Mathlib 中不存在的结果
```

## 失败处理

### 类型 1: 数学 gap
**症状**: Lean 智能体无法填补证明步骤
**处理**: 规划智能体调用非正式智能体生成详细证明

### 类型 2: 复杂度过高
**症状**: 证明项过长，编译超时
**处理**: 规划智能体指示 Lean 智能体分解为子引理

### 类型 3: 形式化路径不可行
**症状**: 依赖 Mathlib 中不存在的基础设施
**处理**: 规划智能体调用非正式智能体提出替代路线

### 类型 4: 重复失败
**症状**: 同一任务失败 3 次以上
**处理**:
1. 规划智能体重新评估整体策略
2. 考虑人工干预
3. 记录到 `~/ai/memory/unresolved/archon-blockers.md`

## 内存持久化

### 每次上下文压缩前
所有智能体写入 `~/ai/workspace/archon/memory/session-<timestamp>.md`:
- 架构性推理
- 失败方法
- 学到的技巧

### 会话结束时
规划智能体写入 `~/ai/workspace/archon/memory/project-<name>-summary.md`:
- 整体进展
- 关键决策
- 未解决问题

## 质量保证

### 编译验证
- 无 `sorry`
- 无 `axiom`
- 无后门

### 语义验证
- 定义与预期数学内容一致
- 定理陈述准确

### 代码质量
- 提取可重用引理
- 移除 `set_option maxHeartbeats`
- 遵循 Mathlib 风格

## 成本优化

1. **优先使用 Gemini**: 数学推理成本低
2. **批量处理**: 并行执行独立任务
3. **早期失败**: 识别不可行路径，快速切换
4. **上下文压缩**: 定期清理，保持聚焦
