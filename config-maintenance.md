# 配置文件维护方法论

适用范围：CLAUDE.md、`.claude/rules/*.md`、模式文件（`~/ai/config/modes/`）、社交配置（`~/ai/config/social/`）的任何新增或修改。

## 工作流（按变更规模分级）

### 分级判断（每次改动前先判断）

| 级别 | 条件 | 流程 |
|------|------|------|
| 轻量 | 单文件、≤3 行改动、不涉及指针变更 | B → C.1（重读确认） |
| 中量 | 单文件 4-15 行、无结构变更（不新建/删除文件） | A.2（SSOT 预检）→ B → C.1（重读）→ C.3（行数） |
| 标准 | 单文件多行 或 涉及指针变更 | A → B → C（全部） → D |
| 重型 | 多文件改动 或 结构性变更（新建/删除文件、改架构） | A → B → C（全部）→ D + 全局 SSOT 审计 |

### A — Pre-mortem（改之前，只分析不动手）

1. **写入目标判定**：适用所有模式？→ `.claude/rules/behavior.md`。单模式？→ 对应模式文件。子领域？→ 子文件并在模式文件登记。经验教训写对应模式文件，不写 `.claude/rules/`
2. **SSOT 预检**：要写入的内容是否已存在于其他文件？已存在 → 用指针（"见 X 文件"），不复制
3. **行数预检**：目标文件当前行数 + 预计新增。CLAUDE.md 仅作索引（<30 行），通用规则在 `.claude/rules/`
4. **影响面**：改动会不会让其他文件的指针/引用失效？列出受影响文件

### B — 执行

- 最小改动，不顺手重构不相关内容
- 新规则引用已有规则用指针，不重述
- `.claude/rules/` 中通用规则文件（behavior.md 等）不复制模式文件的特有内容

### C — 验证（每条都做）

1. 重读修改后的文件，确认内容正确（不靠记忆）
2. 关键词 grep，确认无跨文件重复
3. 行数确认
4. 指针有效性：改了被引用内容 → 检查引用方是否需同步更新

### D — 报告

```
配置变更报告
├─ 文件: xxx.md (行数变化)
├─ SSOT: ✅ 无重复 / ⚠️ 发现重复已修复
├─ 行数: ✅ 在限制内 / ⚠️ 接近上限
├─ 指针: ✅ 全部有效 / ⚠️ 已更新 N 处
└─ 状态: ✅ / ❌
```

## 已知坑（从实际错误中提炼）

| 坑 | 触发场景 | 防御 |
|----|---------|------|
| 指针链超过 2 层 | A→B→C→D 的引用链 | 最多：自动加载→模式文件→子文件。再深 AI 会跟丢，实测已验证 |
| `.claude/rules/` 文件写成模式文件副本 | 创建或编辑 rules 文件时 | 通用规则放 behavior.md，模式特有内容留模式文件 |
| 上下文压缩后编辑引入错误 | compact 后继续改配置 | 编辑前必须 Read 原文件 |
| CLAUDE.md 堆积非索引内容 | 随手往 CLAUDE.md 加规则 | CLAUDE.md 仅作索引，通用规则写 `.claude/rules/behavior.md` |
| 指针目标被修改但引用方未更新 | 重构被引用文件时 | 阶段 C 第 4 步 |
| 同一类问题反复出现但没有规则 | 第三次遇到同类问题时 | 提炼为强制规则写入对应模式文件 |
| 冲突解决规则跨文件重复表述 | 多个文件都写冲突规则 | 通用规则只在 `.claude/rules/behavior.md`，子文件只放本模式特例 |
| config.md globs 路径与实际目录结构不匹配 | 目录重组后忘记更新 globs | 重组后立即验证 globs 能匹配到实际文件 |

## 审计触发条件

- 大批量修改后（≥3 个文件同时改动）
- 上下文压缩后继续编辑时
- 新会话首次编辑配置文件前（快速扫一遍当前状态）
